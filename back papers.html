Computer Architecture & Organization • 70 Marks

Open Full Visual Solutions
Q1 & Q2CPU Registers & Instruction Cycle
  • Registers: GPRs (R0–Rn) store operands; Special: PC (next instruction), IR (current), MAR (address bus), MDR (data bus).
  • Instruction Cycle: Fetch → Decode → Execute → Store (Write-Back). PC increments after fetch.
  • Addressing Modes: Direct, Indirect (pointer), Register (fastest), Indexed (EA = Base + Index).
Q3 & Q4Number Representation & Arithmetic
  • Signed Numbers: Sign-Magnitude (two zeros) → 1's Complement (flip bits) → 2's Complement (flip+1, modern standard).
  • Shift-and-Add Multiplication: Check LSB of multiplier → Add multiplicand if 1 → Shift AQ right. For signed: Booth's Algorithm.
  • Floating Point (IEEE 754): 1 Sign + 8 Exponent + 23 Mantissa bits. Add/Sub: align exponents first. Mul: add exponents, multiply mantissas.
  • Ripple Carry Adder: Chain of Full Adders. Subtraction = 1's complement of B + Cin=1.
Q5 & Q6Memory Organization & Cache
  • Memory Hierarchy: Registers (~1ns) → Cache SRAM (~5ns) → Main RAM (~50ns) → Disk (~5ms). Higher = faster + smaller.
  • Cache Role: Exploits Temporal & Spatial Locality. On miss: fetches entire Block from RAM.
  • Mapping: Direct (1 fixed slot), Fully Associative (any slot), Set-Associative (any slot within a set — best balance).
  • Interleaving: Memory split into banks; CPU fetches from all banks concurrently to maximize bandwidth.
Q7 & Q8Peripheral Devices & I/O
  • Program-Controlled (Polling): CPU busy-loops checking device status — wastes CPU cycles.
  • Interrupt-Driven: Device signals CPU only when ready — CPU free to do other work between transfers.
  • DMA: Dedicated controller handles bulk memory↔device transfers without CPU; raises interrupt only on completion.
  • Software Interrupts: System calls (INT 21h) allow user programs to request OS services.
