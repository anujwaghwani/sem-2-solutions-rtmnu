Basic Blocks & ISA

CPU Subsystems, Execution Cycles & CPU Case Studies

Basic Functional Blocks of a Computer Exam Imp

Based on the Von Neumann Architecture: Shared memory for data and instructions.

🎯 Exam Focus: Frequently asked as a direct 9-10 mark question: "Explain the basic functional blocks of a computer system, including CPU, Memory, I/O, Subsystems, and Control Unit." Be sure to draw the diagram below.

1. CPU (ALU, Registers, CU)

ALU computes arithmetic/logic. Control Unit (CU) coordinates components by fetching and decoding instructions. Registers are ultra-fast local storage (PC, IR, MAR, MDR).

2. Memory Subsystem

Stores active programs and data. The CPU fetches instructions directly from here over the System Bus.

3. I/O Subsystems

Interfaces the digital computer with the external world (keyboards, monitors, network cards).

Instruction Set & Execution Exam Imp

The sequence of operations a CPU performs to execute a single machine instruction. The ISA dictates the registers, cycle, and addressing.

Control Unit & Instruction Formats Exam Imp

Control Unit (CU)

Directs the operation of the processor by generating timing and control signals. It can be implemented as Hardwired (fast, fixed logic gates, typical in RISC) or Microprogrammed (flexible, uses a ROM to store micro-instructions, typical in CISC).

Instruction Formats

Classified by the number of operands explicitly specified in the instruction:

  • 3-Address: ADD R1, R2, R3 (R1 ← R2 + R3). Needs large instruction word, but prevents overwriting source data. (RISC)
  • 2-Address: ADD R1, R2 (R1 ← R1 + R2). One operand acts as both source and destination. (x86)
  • 1-Address: ADD X (ACC ← ACC + X). Uses a single implicit Accumulator (ACC) register.
  • 0-Address: ADD. Uses a Stack (pops top two elements, adds them, pushes result back).

Case Study: CISC (Intel x86)

Complex instructions, variable length. Emphasizes hardware complexity. Can perform math operations directly on memory locations (e.g., ADD [mem], R1). Results in smaller code size but higher CPI.

Case Study: RISC (ARM / MIPS)

Simple, fixed-length instructions. Emphasizes software/compiler complexity. Uses a strict Load/Store architecture (math only happens on registers). Ideal for deep pipelining to achieve CPI of 1.

II

Data Rep & Computer Arithmetic

Integer/Floating Formats, Adders & Multiplication

Data Representation Exam Imp

🎯 Exam Focus: "Explain how characters are represented in computers using ASCII and Unicode" OR "Describe the difference between fixed-point and floating point representations."

Character Encoding

ASCII: 7/8-bit code for English characters. Very limited globally.

Unicode: Universal standard mapping every character globally (UTF-8/16). Solves ASCII limitations.

Integer Representation

Sign-Mag & 1's Comp: Flawed because they have two representations for zero (+0 and -0).

2's Complement: Flip bits + 1. Standard because it has one zero and subtraction uses standard addition circuits.

Fixed vs Floating Point

Fixed Point: Radix point is locked. Fast, simple hardware, but limited dynamic range.

Floating Point: Radix floats. Offers massive dynamic range (IEEE 754 format) at the cost of precision and complexity.

Floating Point Arithmetic (IEEE 754 Standard) Exam Imp

Standard method to represent a massive dynamic range of real numbers.

FP Addition/Subtraction Rules:
  1. Align Exponents: Shift the mantissa of the number with the smaller exponent to the right until exponents match.
  2. Add/Subtract: Perform the operation on the aligned mantissas.
  3. Normalize: Shift the result mantissa and adjust the exponent so the result is back in `1.XXXX` format.

Adder Architectures

Ripple Carry Adder

Chain of Full Adders. Flaw: Very slow. MSB cannot compute until the carry bit "ripples" through all previous stages ($O(n)$ delay).

Carry Look-Ahead Adder (CLA)

Calculates all carry bits in advance using Generate ($G = A \cdot B$) and Propagate ($P = A \oplus B$) logic. Vastly faster but requires complex hardware.

Division Techniques Exam Imp

🎯 Exam Focus: "Compare and contrast the non-restoring and restoring division techniques used in binary division."

Both use Shift-Left and Subtract. How they handle negative remainders differs:

Feature Restoring Non-Restoring
If Remainder < 0 Q=0. Restore (Add Divisor back). Q=0. No Restore. Add divisor in next step.
Efficiency Slower (Subtract + Add per bit) Faster (1 op per bit cycle)

Multiplication Algorithms Exam Imp

🎯 Exam Focus: "Explain Booth's Algorithm..." OR "Discuss shift-and-add and carry save multipliers."

1. Shift-and-Add

The basic approach. Checks multiplier bits: if 1, add multiplicand to accumulator and shift; if 0, just shift. Slow for large numbers.

2. Booth's Multiplier

Optimizes signed math by substituting additions for strings of 1s (e.g., $01110 \rightarrow 2^4 - 2^1$). Uses $Q_0$ and $Q_{-1}$.

3. Carry Save Multiplier

Used for fast hardware multiplication. Uses a tree of Carry Save Adders (CSAs) to add partial products simultaneously without waiting for carry ripples.

III

Memory Organization

Hierarchy, Cache Mapping & Interleaving

Hierarchical Memory Concept

Built upon Locality of Reference (Temporal & Spatial). Hardware leverages small fast memory to hide the latency of large slow memory.

Cache Memory & Mapping Exam Imp

🎯 Exam Focus: "Discuss different types of cache mapping techniques" OR "Discuss the relationship between cache size and block size."

Cache Size vs Block Size

Larger cache = lower miss rate. Larger Block Size exploits Spatial Locality, but if too large, it reduces the total number of blocks the cache can hold, drastically increasing Conflict Misses and Miss Penalty time.

Mapping Functions

  • Direct Mapped: Block $N$ maps to Line $(N \mod \text{CacheSize})$. Fast to check, but suffers from high conflict misses.
  • Fully Associative: Block goes anywhere. Zero conflicts, but needs expensive parallel Tag comparators.
  • Set-Associative: Cache is divided into Sets. Block maps to specific Set, but can occupy any line within it. Provides the best balance of speed and hit rate.

Write Policies

  • Write-Through: Update Cache + Main Memory simultaneously. Keeps memory consistent but is very slow.
  • Write-Back: Update Cache only. Memory updated when block is evicted (using a Dirty Bit). Much faster.

Memory Interleaving Exam Imp

🎯 Exam Focus: "Explain the concept of memory interleaving. Discuss its significance in improving memory access speed."

Overcomes main memory latency by physically dividing memory into concurrent banks. CPU can issue multiple requests across banks simultaneously.

Cache Replacement Algorithms Exam Imp

🎯 Exam Focus: "Explain the working of algorithms like LRU, FIFO, and Optimal Page replacement."

When Memory/Cache is full, algorithms decide which block/page to evict to make room for new data.

FIFO (First-In-First-Out)

Evicts the oldest page. Easy to implement. Flaw: Suffers from Belady's Anomaly (increasing memory can cause MORE page faults).

LRU (Least Recently Used)

Evicts page unused for the longest time (past predicting future). High performance, but complex (requires hardware timers/tracking bits).

Optimal (OPT)

Evicts the page that won't be used for the longest time in the future. Impossible to build; used as a theoretical benchmark.

IV

Peripheral Devices & I/O

DMA, Software Interrupts & Process Scheduling

Peripheral Devices (Input/Output) Exam Imp

🎯 Exam Focus: "Describe different types of peripheral devices and their characteristics. Explain distinction between input and output providing examples."

External hardware components connected to a computer to expand its capabilities.

Input Devices

Convert human actions or physical data into digital binary data.

  • Keyboard: Character input (Electro-mechanical).
  • Mouse: Spatial GUI control (Optical/Laser).
  • Microphone: Audio input (Analog-to-Digital Converter).

Output Devices

Translate digital binary data back into human-understandable form.

  • Monitor: Visual display (LCD/OLED matrix).
  • Printer: Hard copy generation (Laser/Inkjet).
  • Speakers: Audio output (Digital-to-Analog Converter).

I/O Transfers & DMA Subsystem Exam Imp

🎯 Exam Focus: "Explain how DMA works for data transfer between peripherals and memory without CPU intervention."

1. Program Controlled I/O

CPU constantly polls the device. Wastes CPU cycles.

2. Interrupt-Driven I/O

CPU does other work until I/O sends a hardware interrupt. Better, but CPU still moves data word-by-word.

3. Direct Memory Access (DMA)

DMAC chip takes control of System Bus and transfers blocks directly between I/O and Memory. CPU is bypassed.

Software Interrupts, Exceptions & Process Transitions Exam Imp

🎯 Exam Focus: Guaranteed questions. "Explain concept of Software interrupts and exceptions with examples" OR "Explain how interrupts are used in process scheduling."

Exceptions (Traps)

Unplanned, synchronous errors caused by instructions (Divide by zero, Page fault). OS handles these to prevent system failure.

Software Interrupts

Intentional System Calls (e.g., File read). Safely transitions CPU from User Mode to Kernel Mode.

Privileged Instructions

Instructions only executable in OS Kernel Mode (e.g., halting CPU, managing memory). Protects system from malicious user code.

Role of Interrupts in Process State Transitions

Multitasking OS uses a hardware clock generating a Timer Interrupt every few milliseconds. This forces the CPU to pause the running program and jump to the OS Scheduler. The OS performs a Context Switch (saves current state, loads next state), enabling concurrent processes.
