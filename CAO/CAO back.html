1. (A) Discuss the role of registers in a CPU. Differentiate between the types of registers used in instruction execution.

9 Marks

Role of Registers: Registers are the fastest, smallest, and highest-tier memory locations located directly inside the CPU. Their role is to temporarily hold data, memory addresses, and instructions currently being processed by the ALU or Control Unit, avoiding the massive delay of constantly querying main memory.

  • General Purpose Registers (GPRs): (e.g., R0-R31, Accumulator). Available to the programmer/compiler. They store temporary operand data, intermediate math results, or base addresses used during instruction execution.
  • Special Purpose Registers: Strictly managed by the CPU hardware to control execution flow:
    • PC (Program Counter): Holds the memory address of the next instruction to be fetched.
    • IR (Instruction Register): Holds the actual instruction currently being decoded and executed.
    • MAR (Memory Address Register): Connected to the Address Bus; holds the address of the memory block being accessed.
    • MDR/MBR (Memory Data Register): Connected to the Data Bus; acts as a buffer for data flowing into or out of the CPU.

1. (B) Explain the classification of instructions and provide examples of how different types of instructions are executed.

9 Marks

The Instruction Set of a CPU classifies operations into three primary categories based on their function:

  • Data Transfer Instructions: Responsible for moving data between memory and CPU registers, or between registers themselves, without altering the data. Execution: For a LOAD, the CPU places the memory address into the MAR, asserts the "Read" control signal, captures the data in the MDR, and finally moves it to the target register.
  • Data Processing (Arithmetic/Logic): Instructions that perform computation. Execution: For an ADD, the CPU fetches the operands from the source registers, feeds them into the ALU, triggers the ALU's addition circuit, and routes the output back to the destination register.
  • Program Control Instructions: Alter the sequential execution flow of the program (loops, if-statements, function calls). Execution: For a JMP (Jump) or BEQ (Branch if Equal), the CPU evaluates condition flags (if any), and overrides the Program Counter (PC) with a new target address, forcing the next "Fetch" to happen from the new location.

2. (A) Explain the instruction execution cycle in a CPU. Describe each phase of the cycle, including fetch, decode, execute, and store, with examples.

9 Marks

The CPU continually repeats a fundamental operational sequence known as the Instruction Cycle to process a program:

  • 1. FETCH: The Control Unit uses the address in the Program Counter (PC) to retrieve the next instruction from memory. The instruction is loaded into the Instruction Register (IR), and the PC is incremented.
  • 2. DECODE: The Control Unit interprets the binary code inside the IR. It extracts the Opcode (what to do, e.g., ADD) and identifies the addresses or registers of the required operands.
  • 3. EXECUTE: The hardware physically performs the operation. Example: For an ADD R3, R1, R2 instruction, the ALU receives the values from R1 and R2, activates its addition circuit, and calculates the sum.
  • 4. STORE (Write-Back): The result calculated by the ALU in the execute phase is saved to its final destination, which may be a CPU register (like R3 in the example) or a Main Memory location.

2. (B) Discuss the significance of each mode, including direct, indirect, indexed, and register addressing, with examples.

9 Marks

Addressing modes define how the CPU calculates the "Effective Address" (EA) of an operand. Different tasks require different addressing strategies:

  • Direct Mode: Instruction contains the exact memory address (e.g., LOAD 1000). Significance: Simple. Used for accessing static global variables whose locations do not change.
  • Indirect Mode: Instruction holds an address that contains a Pointer to the real data (e.g., LOAD (2000)). Significance: Essential for implementing Pointers. It overcomes instruction-length limits, allowing access to a massive memory space.
  • Register Mode: Operand is held inside a CPU register (e.g., ADD R1, R2). Significance: The absolute fastest mode since it completely avoids slow main memory fetches. Compilers use it heavily for local variables.
  • Indexed Mode: Calculates EA by adding a constant Base Address to a dynamic Index Register (e.g., LOAD 10(R2)). Significance: Mandatory for efficiently traversing Arrays. The base address points to the start of the array, and the index register acts as the loop counter `i`.

3. (A) Explain the concept of signed number representation. Discuss the different methods used for representing signed numbers in binary.

8 Marks

Standard binary only represents positive integers. To handle negative numbers, computers must adopt a specific signed representation method. The diagram shows the three historical methods representing `-5`:

  • Sign-Magnitude: The Most Significant Bit (MSB) acts exclusively as the sign flag (`0` for +, `1` for -). The remaining bits represent the absolute value (magnitude). Flaw: It creates two different bit patterns for zero (+0 and -0) and makes hardware arithmetic circuits highly complex.
  • 1's Complement: A negative number is formed by taking the positive binary number and flipping every bit (`0`s become `1`s). Flaw: While easier to build adders for, it still suffers from the "two zeroes" problem.
  • 2's Complement: The universal modern standard. Formed by finding the 1's complement and mathematically adding `1`. Significance: It perfectly solves the dual-zero issue. Crucially, it allows subtraction to be performed using the exact same ALU adder circuitry as addition (i.e., $A - B$ is processed as $A + (-B)$).

3. (B) Describe the shift-and-add method for multiplication. Explain how this technique works for both unsigned and signed binary numbers.

9 Marks

The Shift-and-Add method mimics how humans perform long multiplication, translating it into hardware components:

  • Unsigned Setup: The Multiplicand is placed in register `M`, Multiplier in `Q`, and Accumulator `A` is initialized to 0. Registers `A` and `Q` act as one continuous shift register.
  • The Algorithm Loop:
    1. The Control Logic inspects the Least Significant Bit of the multiplier ($Q_0$).
    2. If $Q_0 = 1$, the ALU Adds `M` to `A` ($A \leftarrow A + M$). If $Q_0 = 0$, it skips the addition.
    3. Regardless of addition, the combined register `AQ` is Shifted Right by 1 bit. This loop repeats `n` times (for an n-bit multiplier). The final 2n-bit product resides in `AQ`.
  • Handling Signed Numbers: Basic Shift-and-Add fails for negative (2's complement) numbers because shifting right alters the negative magnitude incorrectly. To handle signed numbers, CPUs employ Booth's Algorithm, an advanced variation of shift-and-add that substitutes additions and subtractions based on transitions in the multiplier bits (1 to 0, or 0 to 1), while utilizing an Arithmetic Shift Right to perfectly preserve the sign bit.

4. (A) Discuss how floating-point numbers are represented and how arithmetic operations like addition, subtraction, multiplication, and division are performed.

8 Marks

Floating-point numbers are represented using the IEEE 754 standard, consisting of a Sign bit, a Biased Exponent, and a normalized Mantissa. Arithmetic is complex because the radix point is not fixed:

  • Addition & Subtraction: You cannot directly add two floating-point numbers unless their decimal points align.
    • Align: Take the number with the smaller exponent and shift its mantissa to the right until the exponents are equal.
    • Compute: Add or subtract the aligned mantissas.
    • Normalize: Shift the result mantissa so it returns to standard `1.XXXX` form and adjust the exponent accordingly.
  • Multiplication: Simpler than addition regarding alignment. Multiply the two mantissas together. Add the two biased exponents (and subtract the extra bias of 127). The sign bit is generated using an XOR operation (if signs differ, result is negative). Normalize the final product.
  • Division: Divide the mantissas. Subtract the divisor's exponent from the dividend's exponent (and add back the bias of 127). Determine the sign via XOR and normalize.

4. (B) Explain the process of integer addition and subtraction in binary. Discuss the use of a ripple carry adder and how it performs these operations.

9 Marks

Integer addition and subtraction in modern CPUs rely heavily on the 2's complement representation. The foundational hardware circuit for this is the Ripple Carry Adder (RCA).

  • Structure: As diagrammed, an RCA is constructed by chaining together multiple 1-bit Full Adders (FA). A full adder takes three inputs (A bit, B bit, and a Carry-In bit) and outputs a Sum bit and a Carry-Out bit.
  • Addition Process: The adder calculates $A + B$. The crucial mechanism is that the Carry-Out of FA 0 (the Least Significant Bit) is physically wired into the Carry-In of FA 1. Thus, the carry bit must mathematically "ripple" sequentially down the line from right to left before FA 3 can compute its final sum. This $O(n)$ delay is the RCA's main disadvantage.
  • Subtraction Process: Subtraction ($A - B$) is performed on this exact same addition circuit. Hardware takes the 1's complement of B (by passing it through NOT gates/XORs), and forcibly sets the initial $C_{in}$ of FA 0 to `1`. Mathematically, taking the 1's complement and adding `1` creates the 2's complement. The RCA then just performs standard addition: $A + (-B)$.

Memory Organization

Hierarchy, Cache Mapping & Improving Access Speed

5. (A) Discuss the significance of improving memory access speed and how it is implemented in a system.

9 Marks

Significance: Due to Moore's Law, CPU execution speeds have increased at a vastly higher rate than RAM chip speeds. If memory access speed is not improved, a high-performance CPU will constantly "stall" and waste cycles waiting for data to arrive. This processor-memory performance gap is known as the "Memory Wall."

Implementation: Systems improve access speed through architectural techniques rather than just better chips. The primary implementation is Memory Interleaving (diagrammed above):

  • Main memory is physically divided into multiple independent modules (Banks 0-3). Memory addresses are mapped across these banks (e.g., addr 0 is Bank 0, addr 1 is Bank 1).
  • When a CPU fetches sequential instructions, it doesn't wait for Bank 0 to finish its slow read cycle before asking for the next piece of data. It issues requests to all banks concurrently, heavily pipelining the memory access and drastically increasing effective bandwidth.

5. (B) Explain the role of cache memory in a computer system. Discuss how it bridges the gap between the fast CPU and slower main memory.

9 Marks

The Role of Cache: Cache memory is a small, ultra-fast block of SRAM positioned between the CPU and the main memory (DRAM). Its sole purpose is to act as a high-speed buffer, storing active segments of programs and data so the CPU doesn't have to wait for the slower main memory.

How it bridges the gap: The effectiveness of cache relies entirely on the Principle of Locality:

  • Temporal Locality: If the CPU requests a specific data word, it is highly likely to request that exact same word again soon (e.g., loop variables). The cache keeps a copy of this word, resulting in a fast "Cache Hit" on subsequent requests.
  • Spatial Locality: If the CPU requests a data word, it is likely to request the adjacent words next (e.g., sequential instructions or arrays). To bridge the gap, when a "Cache Miss" occurs, the cache doesn't just fetch one word from RAM; it fetches an entire Block of adjacent data (shown by the green arrow), pre-loading what the CPU will likely need next.

6. (A) Explain the concept of mapping functions used in cache memory. Discuss different types of cache mapping techniques.

9 Marks

Concept of Mapping: Main memory is massively larger than Cache memory. Mapping functions are the strict mathematical rules dictating exactly how and where a large main memory block is allowed to be placed within the limited cache array, and how the CPU locates it later using a Tag.

  • Direct Mapped: A memory block can only map to exactly one specific line (determined by $Line = BlockNumber \mod TotalCacheLines$). It is extremely fast to search (only 1 hardware tag comparison), but heavily prone to "conflict misses" if two active memory blocks map to the same exact slot.
  • Fully Associative: A memory block can be placed in any available line in the cache. This completely eliminates conflict misses since it goes anywhere, but finding the data requires massive amounts of slow, expensive hardware to search every single tag simultaneously.
  • Set-Associative: The modern compromise. The cache is divided into "Sets" (e.g., 2 lines per set). A memory block maps to one specific Set using modulo arithmetic, but can occupy any line within that set. It perfectly balances the low hardware cost of Direct Mapping with the high hit-rate of Associative Mapping.

6. (B) Describe the concept of hierarchical memory organization. Explain the different levels of memory hierarchy (registers, cache, main memory, secondary storage) and their roles in improving system performance.

9 Marks

Concept: It is technologically impossible to build a memory that is simultaneously enormous, lightning-fast, and cheap. Computer architects solve this by creating a hierarchy, illustrated by the pyramid. By combining different memory technologies, the system achieves an overall speed close to the fastest memory, but at a cost-per-byte close to the cheapest memory.

  • Registers: Located directly inside the CPU. They are the absolute fastest storage available (operating at CPU clock speed) but are incredibly small and expensive. Used to hold the exact operands currently being executed by the ALU.
  • Cache Memory: Made of fast SRAM. Sits between the CPU and RAM. Its role is to buffer frequently and recently accessed data blocks, hiding the slow latency of main RAM based on Locality of Reference.
  • Main Memory (RAM): Made of cheaper DRAM. Large capacity. Its role is to store all the active programs and data currently being run by the Operating System.
  • Secondary Storage (Disk): Magnetic HDD or Flash SSD. Extremely slow but provides massive, non-volatile (permanent) capacity. Acts as the permanent repository for files and acts as Virtual Memory when RAM is full.

8 Marks
Feature Program-Controlled I/O (Polling) Interrupt-Driven I/O

7. (B) Explain the concept of software interrupts and exceptions. Illustrate with examples how software interrupts and exceptions are used in modern operating systems.

9 Marks

Unlike hardware interrupts generated by external peripheral devices, software interrupts and exceptions originate synchronously from within the executing CPU instruction itself. The diagram illustrates how they act as critical gateways, forcing the CPU to transition from restricted User Mode to privileged Kernel Mode.

  • Exceptions (Traps/Faults): These are unplanned errors that occur during the execution of an instruction.
    Examples in Modern OS: A program dividing a number by zero, or a "Page Fault" where the CPU tries to access unmapped virtual memory. The hardware catches the trap and forces the OS to handle it gracefully (e.g., terminating the crashing app or fetching the page from disk), preventing a total system failure.
  • Software Interrupts (System Calls): These are intentional, programmed requests made by user applications. Because user programs are isolated, they cannot touch hardware or files directly.
    Examples in Modern OS: If an app needs to print to the screen, read a file, or send network packets, it executes a specific instruction (like INT 0x80 or syscall). This triggers a software interrupt, safely asking the OS kernel to do the privileged work on its behalf.

8. (A) Describe the different types of peripheral devices and their characteristics. Discuss how peripheral devices interact with the Central Processing Unit (CPU) through I/O sub-systems.

8 Marks

Peripheral devices are external hardware components connected to a computer. The CPU operates purely in high-speed digital binary; thus, peripherals cannot connect directly to the CPU. They interact through intermediary I/O Sub-system Modules (Interfaces).

  • Input Devices: Characteristic: Convert human actions or physical data into digital data. Examples: Keyboard, Mouse. Interaction: They send data to an I/O module, which buffers the slow data and fires a hardware interrupt to the CPU when the data is ready to be collected.
  • Output Devices: Characteristic: Translate digital data from the CPU back into human-understandable physical forms. Examples: Monitor, Printer. Interaction: The CPU sends a block of data to the output I/O module, which handles the slow process of converting it into pixels or mechanical print actions independently.
  • Storage (High-Speed): Characteristic: Non-volatile, extremely fast bulk data transfer. Examples: SSDs, HDDs. Interaction: Unlike slow keyboards, disks interact via a DMAC (Direct Memory Access Controller) module, allowing them to bypass the CPU and stream bulk data directly into RAM.

8. (B) Explain how Direct Memory Access (DMA) works for data transfer between peripherals and memory without CPU intervention.

9 Marks

DMA is a critical hardware feature used for bulk data transfer (like fetching a large file from the hard disk), solving the massive inefficiency of forcing the CPU to manually move every single byte of data via registers. By tracking the signals in the diagram:

  • Setup: The CPU programs the dedicated DMA Controller (DMAC) chip's internal registers with the memory start address, the destination, and the total block size (word count), then disconnects to execute other programs.
  • Arbitration: The DMAC asserts the Bus Request (BR) signal. The CPU finishes its current cycle and replies with a Bus Grant (BG) signal, temporarily placing its own bus pins in a high-impedance state to relinquish control of the System Bus to the DMAC.
  • Direct Transfer: As shown by the red dashed arrow, data flows directly from the I/O Disk into Main Memory over the bus, completely bypassing the CPU.
  • Completion: Once the word count hits zero, the DMAC releases the bus and fires a hardware Interrupt to inform the CPU that the transfer is entirely finished.
