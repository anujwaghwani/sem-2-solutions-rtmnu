Q1 (A) Explain the basic functional blocks of a computer system, including the Central Processing Unit (CPU), Memory, Input-Output, Subsystems and Control Unit.

10 Marks

The Von Neumann architecture defines a system where data and instructions are stored in the same read-write memory. The highly detailed diagram illustrates the functional blocks:

  • CPU: The heart of the computer. The Control Unit (CU) acts as the traffic cop, decoding instructions and generating control signals. The ALU computes math and logic. Registers provide instantaneous storage. The Memory Address Register (MAR) holds the address, and the Memory Data Register (MDR) holds the actual data being transferred.
  • System Buses: The components communicate via three dedicated buses. The Address Bus (red, unidirectional) tells memory *where* to look. The Data Bus (green, bidirectional) carries the actual information. The Control Bus (blue) carries Read/Write signals.
  • Memory & I/O: Memory holds active programs. I/O Subsystems translate real-world physical data into machine-readable digital data (Input) and vice versa (Output).

Q1 (B) Define and explain different addressing modes used in a CPU. Discuss the significance of each mode.

10 Marks

Addressing modes define the rules the CPU uses to calculate the Effective Address (EA) of the data required by an instruction. The diagrams explicitly map out the math:

  • Immediate Mode: The operand is not in memory; it is embedded directly inside the instruction itself. Significance: Execution is extremely fast as no memory fetch is required. Primarily used to initialize variables or loop counters (e.g., x = 5).
  • Direct Mode: The instruction contains the exact EA of the data. Significance: Simple to implement. Used for accessing static or global variables whose addresses are known at compile time.
  • Indirect Mode: The instruction contains an address (A), but that address holds a Pointer (B) to the actual data. Significance: Grants tremendous flexibility. It is essential for implementing pointers in languages like C/C++ and allows the CPU to access a much larger memory space than the instruction width permits.
  • Register & Indexed Modes: Register mode keeps data entirely within CPU registers. Indexed mode calculates EA = Base Register + Index, which is mandatory for efficiently traversing Array elements (e.g., Array[i]).

Q2 (A) Discuss the concept of an instruction set in a CPU.

10 Marks

The diagram clearly illustrates that the Instruction Set Architecture (ISA) acts as the crucial boundary and translation bridge between high-level software and physical hardware.

  • Vocabulary of the CPU: The ISA is the definitive list of all fundamental commands (like ADD, LOAD, JUMP) that a specific CPU is physically wired to understand and execute. If a command is not in the ISA, the CPU cannot perform it directly.
  • Hardware Abstraction: As shown, compilers translate high-level code down into assembly code defined by the ISA, and assemblers turn that into machine code (1s and 0s). This hides the complex electronic circuitry from the programmer.
  • Architecture Dependence: ISAs are tied to CPU families. Code compiled for an Intel x86 ISA processor will not run directly on an ARM ISA processor (like in a smartphone) because their "vocabularies" are fundamentally different.

Q2 (B) Explain the different component of Instruction Set architecture, including registers instruction execution cycle and addressing modes.

10 Marks

The Instruction Set Architecture defines the state and processing loop of the machine. The components work together in the detailed cycle shown above:

  • Registers (Programmer State): The ISA defines accessible registers. The execution begins in the FETCH phase, heavily relying on the Program Counter (PC) to locate the instruction and the Instruction Register (IR) to hold it.
  • Addressing Modes & Decoding: In the DECODE stage, the control unit inspects the opcode and uses the specified Addressing Mode rules to resolve the Effective Address (EA) of the data operands.
  • Execute, Store, & Interrupt: The ALU performs the math (EXECUTE), writes the result back (STORE), and crucially, before moving to the next instruction, the CPU checks if any external hardware has fired an INTERRUPT signal.

Q3 (A) Describe the difference between fixed-point and floating point number representations.

10 Marks

The diagrams physically contrast how computers handle real numbers (numbers with fractional components):

  • Fixed-Point: As seen in the top diagram, the radix (decimal) point is permanently locked in hardware (e.g., exactly 16 bits for the integer, 16 for the fraction). Advantage: Arithmetic is identical to integer math, making it very fast and simple. Disadvantage: Extremely limited dynamic range.
  • Floating-Point: As seen in the bottom diagram, the number is split into a Sign, Exponent, and Mantissa. The radix point can "float" dynamically because the exponent shifts its location mathematically (Value = $(-1)^S \times 1.M \times 2^{E-127}$). The diagram shows a 32-bit example. Advantage: Massive dynamic range, capable of representing sub-atomic sizes and cosmic distances. Disadvantage: Slower, requires complex Floating Point Unit (FPU) circuitry.

Q3 (B) Explain Booth's Multiplication algorithm. Discuss how it optimizes the multiplication of signed binary numbers and reduces the number of partial products compared to the traditional shift-and-add method.

10 Marks

Booth's algorithm optimizes signed 2's complement multiplication by mathematically recognizing that sequences of 1s can be converted into a single subtraction and addition. Following the algorithmic flowchart:

  • Initialization: The accumulator ($A$) and a fabricated trailing bit ($Q_{-1}$) are set to 0. The multiplier is $Q$, multiplicand is $M$, and the loop $Count$ is the number of bits.
  • The Logic: The algorithm inspects the current multiplier bit ($Q_0$) and the previous bit ($Q_{-1}$).
    • If it encounters the start of a string of 1s (10), it performs a subtraction ($A - M$).
    • If it encounters the end of a string of 1s (01), it performs an addition ($A + M$).
  • Optimization: As shown by the dashed center path, if it is in the middle of a string of 1s (11) or 0s (00), it skips the arithmetic operation entirely and moves straight to the Arithmetic Shift Right. This massively reduces the number of partial products calculated compared to traditional shift-and-add, making hardware faster.

Q4 (A) Explain how characters are represented in computers. Discuss common encoding schemes such as ASCII and unicode.

10 Marks

Computers process all data fundamentally as binary numbers. To represent text, a computer relies on a Character Encoding Scheme. The diagram details the evolution:

  • ASCII (American Standard Code for Information Interchange): As seen on the left, it uses a rigid 7-bit (or 8-bit extended) format. For example, 01000001 translates to 'A'. Because 8 bits can only generate $2^8 = 256$ unique patterns, ASCII is strictly limited to the English alphabet, digits, and basic control characters. It is inadequate for global computing.
  • Unicode: A universal standard designed to assign a unique number (Code Point) to every character in almost every written language in the world, plus symbols and emojis.
  • UTF-8 Implementation: The diagram on the right shows how Unicode is practically stored in memory using UTF-8. It uses a variable-width encoding. Standard English letters take 1 byte (making it perfectly backwards compatible with ASCII), but complex characters or emojis automatically expand to use up to 4 bytes (e.g., `11110xxx 10xxxxxx...`).

Q4 (B) Compare and contrast the non-restoring and restoring division techniques used in binary division.

10 Marks

Both algorithms utilize a "shift-left and subtract" mathematical loop to perform binary hardware division (Dividend in Q, Divisor in M). They differ entirely in how they handle a "failed" subtraction (when the accumulator `A` becomes negative), as shown in the flow blocks:

  • Restoring Division: If the initial subtraction (`A = A - M`) yields a negative result, it immediately corrects the mistake by adding the divisor back (`A = A + M`, the red RESTORE block). Drawback: It is slower because it potentially requires two arithmetic operations (subtract, then add) in a single cycle if the guess was wrong.
  • Non-Restoring Division: If negative, it skips the redundant restore step entirely (the green block). Instead, it leaves `A` negative, and compensates by making the next cycle an Addition operation rather than a Subtraction. Benefit: Faster execution, guaranteeing exactly one arithmetic operation per bit.

Q5 (A) Explain the concept of memory interleaving. Discuss its significance in improving memory access speed and how it is implemented in a system.

10 Marks

Memory interleaving is an architectural technique that improves memory access speed by overcoming the physical latency limits of RAM chips. As depicted in the diagram:

  • Implementation (Low-Order Interleaving): The memory is physically divided into multiple independent modules (Banks 0-3). As shown at the top, the address is decoded so that the lowest order bits select the bank. This guarantees that sequential memory addresses (like 0, 1, 2, 3) are mapped across different physical banks.
  • Significance: When the CPU executes a program, it fetches sequential instructions. Instead of waiting for Bank 0 to complete its slow read cycle before requesting address 1, the CPU issues requests to all banks concurrently. This heavily pipelines memory access, massively increasing the effective memory bandwidth.

Q5 (B) Discuss the relationship between cache size and block size in a cache memory system.

10 Marks

The relationship between Cache Size and Block Size dictates the efficiency of the cache, forming a distinct U-shaped curve as seen in the graph.

  • Cache Size: Increasing the overall capacity of the cache directly lowers the miss rate, but costs more silicon area and slightly slows down lookup time.
  • Block Size (Line Size): As shown in the array diagram, a block is the minimum unit of data transferred from RAM. As block size increases initially, the Miss Rate Drops because of Spatial Locality (bringing in adjacent data words that the CPU will likely execute next).
  • The Tradeoff: If the block size becomes too large for a fixed cache size, the cache can only hold a few blocks in total. This causes old blocks to be evicted constantly, resulting in a severe spike in Conflict Misses (the right side of the red U-curve). Furthermore, huge block sizes dramatically increase the "miss penalty" latency when fetching from RAM.

Q6 (A) Discuss different types of cache mapping techniques, such as direct mapped, associative and set-associative mapping.

10 Marks

Cache mapping determines exactly how and where a block from main memory finds its place in the cache array. The formulas and arrows in the diagram illustrate the three rules:

  • Direct Mapped: The block can only map to exactly one specific line determined by modulo arithmetic ($Line = MemBlock \mod TotalCacheLines$). It is extremely fast to search (only 1 tag comparison), but heavily prone to "conflict misses" if two active blocks map to the same slot.
  • Fully Associative: The block can map to any available line in the cache. This completely eliminates conflict misses since it goes anywhere, but it requires massive amounts of slow, expensive hardware to search every single tag simultaneously.
  • Set-Associative: The modern compromise. The cache is divided into "Sets" (e.g., 2 lines per set). A memory block maps to one specific Set using modulo arithmetic ($Set = MemBlock \mod TotalSets$), but can occupy any line within that set. It beautifully balances the low hardware cost of Direct Mapping with the high hit-rate of Associative Mapping.

Q6 (B) Explain the working of algorithms like Least Recently Used (LRU), First-in-First-out and Optimal Page Replacement.

10 Marks

When physical memory is full and a Page Fault occurs, the OS must use a Replacement Algorithm to intelligently evict an existing page. The diagram highlights the logic differences using a timeline:

  • FIFO (First-In-First-Out): The simplest method. It tracks pages in a queue and indiscriminately evicts the oldest page (Page A). Drawback: May evict heavily used system variables just because they are old. It remarkably suffers from Belady's Anomaly, where increasing physical memory frames can actually cause MORE page faults.
  • LRU (Least Recently Used): Looks backward at usage history. It evicts the page that has not been accessed for the longest duration in the past (Page B, since A and C were used more recently). Advantage: Excellent practical performance based on the principle of temporal locality. Drawback: Hardware complex (requires tracking bits).
  • Optimal (OPT): The theoretical perfect algorithm. It looks forward into the future instructions and evicts the page that will not be needed for the longest time (Page C). Since predicting the exact future execution path is physically impossible, it is strictly used as an analytical benchmark for comparison.

Q7 (A) Describe the different types of peripheral devices and their characteristics. Explain the distinction between input and output devices, providing examples.

10 Marks

Peripheral devices are external hardware components connected to a computer to expand its capabilities. The diagram distinguishes the data flow direction and characteristics for the two main types:

  • Input Devices (Red Arrows): They convert human actions or physical environmental data into machine-readable digital binary data, feeding it inward to the CPU. Often requires Analog-to-Digital Converters (ADCs). Examples: Keyboards (electro-mechanical character input), Mice (spatial navigation), Scanners, and Microphones.
  • Output Devices (Green Arrows): They take processed digital data from the CPU and translate it outward into human-understandable physical forms. Often requires Digital-to-Analog Converters (DACs). Examples: Monitors (visual pixels), Printers (hardcopy ink/laser), and Speakers (sound waves).

Q7 (B) Explain the concept of Software interrupts and exceptions. Illustrate with examples how software interrupts and exceptions are used in modern operating systems.

10 Marks

Unlike hardware interrupts generated by external peripheral devices, software interrupts and exceptions originate synchronously from within the executing CPU instruction itself. The diagram illustrates how they act as gateways, forcing the CPU to transition from restricted User Mode to privileged Kernel Mode.

  • Exceptions (Traps/Faults): These are unplanned errors that occur during execution. Example: A program dividing a number by zero, or a Page Fault trying to access unmapped memory. The hardware catches the trap and forces the OS to handle it gracefully (via the Vector Table), preventing a total system crash.
  • Software Interrupts (System Calls): These are intentional, planned requests made by user programs. Because user programs are isolated, they cannot touch hardware directly. Example: If a program needs to read a file from the hard drive or open a network port, it executes a specific instruction (like INT 0x80 or syscall) triggering a software trap, safely asking the OS kernel to do the privileged work on its behalf.

Q8 (A) Explain how Direct Memory Access (DMA) works for data transfer between peripherals and memory without CPU intervention.

10 Marks

DMA is a critical hardware feature used for bulk data transfer (like fetching a file from the hard disk), solving the massive inefficiency of the CPU having to manually move every single byte of data. By tracking the signals in the diagram:

  • Setup: The CPU programs the DMA Controller (DMAC) internal registers with the memory start address, the destination, and the block size (word count), then disconnects to do other work.
  • Arbitration: The DMAC asserts the Bus Request (BR) signal. The CPU finishes its current cycle and replies with Bus Grant (BG), temporarily relinquishing control of the System Bus.
  • Direct Transfer: As shown by the red dashed arrow, data flows directly from the I/O Disk into Memory, completely bypassing the CPU. This is done via Burst Mode (moving the whole block at once) or Cycle Stealing (interlacing with CPU cycles).
  • Completion: Once the word count hits zero, the DMAC releases the bus and sends an Interrupt (INT) to inform the CPU it finished.

Q8 (B) Explain how interrupts are used in process scheduling. Discuss the importance of interrupt handling in modern operating systems.

10 Marks

Interrupts are the absolute backbone of a modern Operating System, allowing it to perform preemptive multitasking and intelligent process scheduling. Following the diagram loop:

  • Preemptive Scheduling: A system relies on a hardware clock that fires a Timer Interrupt every few milliseconds (the Time Slice).
  • The Transition: As shown by the red dotted arrow, when this timer interrupts the CPU, the OS forcefully removes the currently "Running" process, seizing control of the CPU back to Kernel mode, and moves the process back to the "Ready" queue.
  • Context Switch: The OS Scheduler's Interrupt Service Routine (ISR) saves the registers (Process Control Block) of the paused process and dispatches the next process from the Ready queue. This high-speed interruption loop creates the illusion that multiple programs are executing simultaneously.
  • Importance: Without interrupt handling, a badly written program entering an infinite loop would completely freeze the entire computer. Furthermore, I/O Interrupts (yellow arrow) optimize CPU usage by waking up "Blocked" processes only once their requested disk/network data is finally ready, avoiding polling.
