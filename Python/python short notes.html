Python Programming

Complete Revision Notes: Units 1 to 5 (Detailed Operations)

UNIT I: Basics, Data Types & I/O

1. Core Features & Architecture

  • Features: Simple, dynamically typed, interpreted, platform-independent, multi-paradigm (OOP & procedural), and huge standard library.
  • Internal Working: Source Code (.py) → Compiler → Bytecode (.pyc) → Python Virtual Machine (PVM) executes it line-by-line.
  • Tokens: Smallest units. Includes Keywords (if), Identifiers (names), Literals (values like 10), Operators (+), Punctuators (:).

2. Data Types & Variables

Python is dynamically typed (no need to declare variable types). Core types:

  • Numeric: int, float, complex (e.g., 2+3j). Immutable.
  • Sequence: str (immutable), list (mutable), tuple (immutable).
  • Mapping/Sets: dict (mutable, key-value), set (mutable, unique).
  • Boolean: bool (True/False).

3. Input, Output & String Operations

  • input(): Always returns a string. Requires typecasting for math: int(input()).
  • eval(): Evaluates a string as Python code. Can automatically detect input types (e.g., eval("[1,2]") returns a list).

String Operations & Slicing: Strings are immutable sequences.

  • Slicing: string[start:stop:step]. "PYTHON"[::-1] reverses the string.
  • Common Methods:

UNIT II: Operators & Control Flow

1. Operators in Detail

  • Arithmetic: +, -, *, / (float div), // (floor div), % (modulus), ** (exponent).
  • Bitwise: Operates on binary. & (AND), | (OR), ^ (XOR), ~ (NOT), << (Left Shift), >> (Right Shift).
  • Identity (is, is not): Checks if two variables point to the exact same memory object.
  • Membership (in, not in): Checks if a sequence contains a specific value.

2. Decision & Loop Control

  • Decision: if, elif, else. Can be nested.
  • while loop: Repeats while a condition is True. Requires manual iterator update.
  • for loop: Iterates over sequences. Often used with range(start, stop, step).
  • Jump Statements:
    - break: Terminates the whole loop immediately.
    - continue: Skips the rest of the current iteration and goes to the next.

UNIT III: Functions and Lists

1. Functions

A reusable block of code defined using the def keyword.

  • Scope: Local variables exist only inside the function. Global variables exist outside and can be accessed anywhere.
  • Return Statement: Terminates the function and passes a value back to the caller. If omitted, returns None.
  • Recursion: A function that calls itself to solve smaller instances of the same problem (requires a base case to stop).
  • Lambda Functions: Anonymous, single-line functions. Syntax: lambda args: expression.

2. List Operations

Lists are ordered, mutable sequences enclosed in brackets []. Can hold mixed data types.

UNIT IV: Tuples, Sets & Dictionaries Operations

1. Tuple Operations

Ordered, immutable sequences. Enclosed in ().

  • Creation: t = (1, 2) or t = 1, 2. Single element requires a comma: t = (10,).
  • Math Ops: Concatenation (1, 2) + (3, 4). Repetition (1, 2) * 2.
  • Methods: Because they are immutable, they only have two methods:
    • count(x): Counts occurrences of x.
    • index(x): Finds first index of x.
  • zip() and Inverse zip(*): Pairs elements from multiple iterables. zip(*) unzips them back.

2. Set Operations

Unordered collections of unique elements. Created using {} or set(). Mutable.

  • Modification Methods:
    add(x): Adds an element.
    remove(x): Removes x (throws error if not found).
    discard(x): Removes x (does nothing if not found).
    clear(): Empties the set.

UNIT V: File Handling Operations

1. Opening and Closing Files

Python uses the built-in open() function. Always close files using file.close() or implicitly using the with statement.

  • "r" (Read): Default. Opens for reading.
  • "w" (Write): Opens for writing. Overwrites existing data or creates a new file.
  • "a" (Append): Opens for writing. Appends data to the end of the file without overwriting.

2. Reading and Writing Methods

MethodDescription
read([size])Reads the entire file into a single string (or up to `size` bytes).
readline()Reads a single line from the file.
readlines()Reads all lines and returns them as a list of strings.
write("string")Writes a single string to the file. (Must cast numbers to string first!)
writelines(list)Writes a list of strings to the file (does not add newlines automatically).

3. Cursor Operations (Seek and Tell)

When you read or write, a "cursor" (file pointer) moves through the file.

  • file.tell(): Returns the current integer position (byte offset) of the cursor within the file.
  • file.seek(offset, whence): Moves the cursor to a specific position.
    - seek(0) moves the cursor to the absolute beginning of the file. Useful if you want to read a file again after writing to it. 
# Writing and Reading Example
with open("data.txt", "w+") as file:
    file.writelines(["Line 1\n", "Line 2\n"])
    
    print("Cursor position after write:", file.tell())
    
    # Move cursor back to beginning to read
    file.seek(0)
    print("File Content:\n" + file.read().strip())
